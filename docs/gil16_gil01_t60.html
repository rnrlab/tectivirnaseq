GIL16 vs GIL01 contrast

Author

Eduardo Lozano

Published

May 22, 2026

To analyze differential expression across strains and time points, we performed all statistical modeling within a single DESeq2 dataset rather than creating separate DESeq2 objects for each time point. Because the experimental design includes strong temporal dynamics associated with phage induction, library composition changes substantially across conditions. Analyzing all samples together allows DESeq2 to estimate size factors and dispersion parameters using the full dataset, thereby borrowing statistical strength across conditions and ensuring a consistent normalization framework. Importantly, replicate effects (treated as a batch factor in the model) are captured globally; splitting the dataset by time would force DESeq2 to re‑estimate normalization and dispersion independently for each subset, disrupting the shared technical structure of the experiment and potentially introducing artificial differences. Using a unified DESeq2 object therefore provides a coherent, statistically robust foundation that preserves biological comparability, controls technical variation and enables accurate strain‑level contrasts at each time point.

In order to perform the GIL16 vs GIL01 contrast at 60 minutes, we established a log₂ fold-change (LFC) of ±0.585. This value reflects an increase or decrease in expression of 50%, depending on whether it is positive or negative, respectively. The default value is 0, but when performing the other contrasts we obtained too many differentially expressed (DE) genes. This violates one of DESeq2’s key assumptions, which is that most genes are not differentially expressed.

Code 
contrast <- results(dds, contrast = list(c("Strain_GIL16_vs_GBJ002", "Time60.StrainGIL16"), c("Strain_GIL01_vs_GBJ002", "Time60.StrainGIL01")), lfcThreshold = log2(1.5))
contrast_df <- as.data.frame(contrast)

contrast_df <- contrast_df %>%
  { x <- .; x$Gene_ID <- rownames(x); x } %>%
  left_join(
    annotation %>%
      filter(Gene_ID %in% rownames(contrast_df)) %>%
      dplyr::select(Gene_ID, Gene_symbol, Description, COG_category, Association, Type, Subtype, Source),
    by = "Gene_ID"
  ) %>%
  rename(
    Mean_normalized_counts = baseMean,
    LFC = log2FoldChange,
    Adjusted_p_value = padj
  ) %>%
  dplyr::select(
    Gene_ID,
    Gene_symbol,
    Description,
    COG_category,
    Association,
    Type,
    Subtype,
    Source,
    Mean_normalized_counts,
    LFC,
    Adjusted_p_value
  )

MA plot

A MA plot visualizes the relationship between the mean expression and the LFC for each gene. It helps to identify genes that are differentially expressed across conditions, highlighting those with large fold changes relative to their expression level.

Code 
plotMA(contrast, alpha = 0.05, xlab = "Mean of normalized counts", ylab = "LFC")

DE genes

DE genes are defined as those with an adjusted p-value below 0.05. Among them, upregulated genes are extracted as those with a LFC > 0.58, while downregulated genes are defined as those with an LFC < -0.58.

Code 
up_genes <- rownames(subset(contrast, padj < 0.05 & log2FoldChange > log2(1.5) & !is.na(padj)))
down_genes <- rownames(subset(contrast, padj < 0.05 & log2FoldChange < -log2(1.5) & !is.na(padj)))

Volcano plot

A volcano plot displays each gene according to its LFC on the x-axis and the statistical significance (−log₁₀ adjusted p-value) on the y-axis. Only the isocitrate lyase (GBJ002_03895) and DUF4153 domain-containing protein (GBJ002_04612) genes were upregulated, i.e., more expressed in the GIL16 samples than in the GIL01 samples.

Code 
volcano_plot(contrast_df, limits = c(-5, 5))

GO

The Gene Ontology (GO) is organized into three categories: Biological Process (BP), which describes the biological objectives to which gene products contribute; Molecular Function (MF), which refers to the elemental activities of gene products at the molecular level; and Cellular Component (CC), which specifies the locations within the cell where gene products are active. In any case, no enrichments were found.

Enrichment summary

This table contains a summary of the enrichments for each category. The subcategory was used for KEGG and the description for GO. Although you can scroll, I recommend exporting using the Excel button and viewing the table in full screen.

Code 
es_up <- enrichment_summary(up_genes, categories = list(cog_up, kegg_up, go_bp_up, go_mf_up, go_cc_up, ec_1_up, ec_2_up, ec_3_up))
es_down <- enrichment_summary(down_genes, categories = list(cog_down, kegg_down, go_bp_down, go_mf_down, go_cc_down, ec_1_down, ec_2_down, ec_3_down))
es_de <- rbind(es_up, es_down)
